Brackets and Things

Parentheses () are used to group and contain expressions and parameters. Parentheses also have multiple uses relating to functions and structures. They can be used to contain a list of parameters passed to functions and control structures and they are used to group expressions to control the order of execution. Some functions have no parameters and in this case, the space between parentheses is blank.

Square brackets [] can be used to set and access variables stored in arrays. Square brackets can also be used to set and access values stored in objects by using bracket notation (e.g, myObj[“keyName”]. Object values can also be set and accessed using dot notation (e.g, myObj.keyName), which does not use any brackets whatsoever.

Braces {} are used to initialise and declare objects. In object literal notation, all of the keys and values are defined between the braces, though these properties and values can be modified later. Braces are also used to define the code in functions and and flow-control elements, such as for loops.

JavaScript allows you to use either double quote (“ ”) or single quote (‘ ‘) marks to create a string. Both methods are valid but I tend to use double quotes most of the time for a number of reasons. First, I am used to double quotes for strings and single quotes for chars from other languages. Second, using double quotes is more forgiving when using apostrophes in strings and no escapes are needed. Third, JSON requires the use of double quotes.
