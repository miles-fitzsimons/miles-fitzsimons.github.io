The time I got blocked on a simple problem and solved it in an elegant way

I got blocked on using an HTML wrapper while building this website and my JavaScript racer.

A wrapper is an element (commonly a div) that encloses one or more other elements in the HTML markup, such as:

<div id="wrap">
  <h1>Headline</h1>
  <p>Paragraph</p>
  <p>Paragraph</p>
</div>

Wrappers have many uses including enabling special positioning of HTML elements. My problem in particular was how to set up the wrapper and then use it for element positioning.

I solved the problem through Googling, trial and error and talking to my peers. I found solving the problem initially challenging but then frustrating – I feel I spent a lot of time achieving nothing. Ultimately, talking to my peers was the most useful problem solving technique.

I learned about some of the subtleties of CSS and the importance of only styling what is needed, when it is needed. This also helped with my understanding of CSS frameworks, such as the Skeleton framework.

Reflection on problem-solving techniques

This week I used a number of problem solving techniques as discussed below.

Pseudocode involves breaking a coding problem down into small parts and writing in plain-ish English what is required to solve each part. This is a useful technique and I use it before writing most of my code. It can help overcome mental barriers involved with looking at one big problem.

Trying something is exactly that and often what I do after pseudo coding. Sometimes it works and sometimes it does not. Trying something else can be useful, but often other problem solving techniques are required.

Error messages, though confusing, can be very useful. For example, often a simple syntax error will throw out your whole code. Error messages usually have an accompanying line number that is useful for identifying where your code is breaking. Beyond this, error messages can give great insight into a problem, though I found myself sometimes needing to use other problem solving techniques to interpret a error message.

Console logging is powerful for when your code works, but not in the manner that you expect. Console logs can be added into code and used to print out, for example, current variable values. This can be useful for figuring out what exactly your code is computing at any given point.

Google is invaluable. Googling your problem usually shows that you are not the first to have it and often provides a solution. It can be hard to Google some problems and often JavaScript queries come back with jQuery solutions. You also usually need to modify the given solution somewhat to adapt it to your particular problem.

Finally, talking to people can be a great help. Other people may approach your problem in a completely different way, and it can be useful to get this alternate perspective.

JavaScript array methods

The .map() method creates a new array with the results of calling a provided function on every element in this array. For example, calling:

var numbers = [1,2,3,4,5];
var squares = numbers.map(function(num){return num*num;});

will return set squares as an array of the squares of the numbers (i.e, squares = [1,4,9,16,25]).


The filter() method creates a new array with all elements that pass the test implemented by the provided function. For example, calling:

var arr = [12, 5, 8, 130, 44];
var filtered = arr.filter(function(num){return num > 10;});

will return filtered as an array of the numbers in arr that are greater than 10 (i.e, filtered = [12,130,44]).


The reduce() method applies a function against an accumulator and each value of the array (from left-to-right) to reduce it to a single value. For example, calling:

var arr=[0,1,2,3,4,5,6];
var total = arr.reduce(function(a, b) {return a+b;});

will return total as an integer equal to the sum of the numbers in arr (i.e, total = 21).
